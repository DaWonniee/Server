/* 실무 개발에 사용되는 package.json 파일에 대한 메모입니다. package.json 파일은 Node.js 프로젝트에서 사용되는 중요한 구성 파일로, 프로젝트의 메타데이터와 의존성 정보를 포함하고 있습니다. 이 파일은 프로젝트의 루트 디렉토리에 위치하며, npm(Node Package Manager)을 사용하여 패키지를 설치하고 관리하는 데 사용됩니다. package.json 파일의 주요 구성 요소는 다음과 같습니다: 1. name: 프로젝트의 이름을 지정합니다. 일반적으로 소문자와 하이픈(-)을 사용하여 작성됩니다. 2. version: 프로젝트의 버전을 지정합니다. 일반적으로 SemVer(유Semantic Versioning) 형식을 따릅니다. 3. description: 프로젝트에 대한 간단한 설명을 제공합니다. 4. main: 프로젝트의 진입점 파일을 지정합니다. 일반적으로 index.js 파일이 사용됩니다. 5. scripts: 프로젝트에서 사용할 수 있는 명령어 스크립트를 정의합니다. 예를 들어, "start", "test", "build" 등의 스크립트를 정의할 수 있습니다. 6. dependencies: 프로젝트에서 필요한 외부 패키지와 그 버전을 지정합니다. 이 패키지들은 프로젝트 실행 시 필요합니다. 7. devDependencies: 개발 환경에서만 필요한 패키지와 그 버전을 지정합니다. 예를 들어, 테스트 프레임워크나 빌드 도구 등이 포함될 수 있습니다. 8. keywords: 프로젝트와 관련된 키워드를 배열로 지정합니다. 검색 엔진에서 프로젝트를 찾는 데 도움이 됩니다. 9. author: 프로젝트의 작성자 정보를 지정합니다. 10. license: 프로젝트의 라이선스를 지정합니다. 예를 들어, "MIT", "ISC" 등이 있습니다. package.json 파일은 npm을 사용하여 패키지를 설치할 때 자동으로 생성되거나, 개발자가 직접 작성할 수도 있습니다. 이 파일을 통해 프로젝트의 의존성을 관리하고, 다른 개발자와 협업할 때 필요한 정보를 제공할 수 있습니다. npm i 만 입력하면 package.json 파일에 정의된 dependencies와 devDependencies에 따라 필요한 패키지들이 설치됩니다. package.json 에 주로 포함되는 구성 : { "name": "my-project", "version": "1.0.0", "description": "A sample Node.js project", "main": "index.js", "scripts": { "start": "node index.js", "test": "mocha" }, "dependencies": { "express": "^4.17.1", "mongoose": "^5.12.3" }, "devDependencies": { "mocha": "^8.3.2", "chai": "^4.3.4" }, "keywords": ["nodejs", "express", "mongodb"], "author": "Your Name", "license": "MIT" } * 추가로 알면 좋은 다운로드 패키지 이름들 - express : Node.js에서 웹 애플리케이션을 개발하기 위한 프레임워크로, 라우팅, 미들웨어, 요청/응답 처리 등을 간편하게 구현할 수 있습니다. - mongoose : MongoDB와 상호작용하기 위한 ODM(Object Data Modeling) 라이브러리로, 스키마 정의, 데이터 검증, 쿼리 빌딩 등을 지원합니다. - nodemailer : 이메일 전송을 위한 패키지로, SMTP 서버를 통해 이메일을 보내거나 수신할 수 있습니다. - axios : HTTP 요청을 보내고 응답을 처리하기 위한 패키지로, RESTful API와 통신할 때 자주 사용됩니다. - dotenv : 환경 변수를 관리하기 위한 패키지로, .env 파일을 사용하여 민감한 정보를 안전하게 저장하고 불러올 수 있습니다. - bcrypt : 비밀번호 해싱을 위한 패키지로, 사용자 비밀번호를 안전하게 저장하고 검증하는 데 사용됩니다. - jsonwebtoken : JWT(JSON Web Token)를 생성하고 검증하는 데 사용되는 패키지로, 인증 및 권한 부여에 활용됩니다. - cors : Cross-Origin Resource Sharing(CORS)을 처리하기 위한 패키지로, 다른 도메인에서의 요청을 허용하거나 제한하는 데 사용됩니다. - express-validator : 요청 데이터의 유효성을 검사하고 검증하는 데 사용되는 패키지로, 입력 값의 형식이나 조건을 확인할 수 있습니다. - morgan : HTTP 요청 로깅을 위한 미들웨어 패키지로, 서버에 들어오는 요청 정보를 기록하고 분석하는 데 사용됩니다. - multer : 파일 업로드를 처리하기 위한 미들웨어 패키지로, 클라이언트에서 전송된 파일을 서버에서 처리하고 저장할 수 있습니다. - nodemon : 개발 중에 서버를 자동으로 재시작해주는 패키지로, 코드 변경 시 서버를 수동으로 재시작할 필요 없이 편리하게 개발할 수 있습니다. - body-parser : 요청 본문(body)을 파싱하여 JSON이나 URL-encoded 데이터를 처리하는 미들웨어 패키지로, 클라이언트에서 전송된 데이터를 쉽게 다룰 수 있습니다. - helmet : 보안 관련 HTTP 헤더를 설정하여 애플리케이션을 보호하는 미들웨어 패키지로, XSS, 클릭재킹 등의 공격으로부터 서버를 보호할 수 있습니다. - compression : HTTP 응답을 압축하여 전송하는 미들웨어 패키지로, 네트워크 대역폭을 절약하고 응답 속도를 향상시킬 수 있습니다. - express-session : 세션 관리를 위한 미들웨어 패키지로, 사용자 세션을 생성하고 유지하는 데 사용됩니다. - cookie-parser : 쿠키를 파싱하고 처리하는 미들웨어 패키지로, 클라이언트에서 전송된 쿠키를 쉽게 다룰 수 있습니다. - express-rate-limit : 요청 속도를 제한하여 서버를 보호하는 미들웨어 패키지로, DoS(Denial of Service) 공격을 방지할 수 있습니다. - express-async-errors : 비동기 함수에서 발생한 오류를 처리하기 위한 패키지로, Express에서 비동기 함수의 오류를 자동으로 처리할 수 있습니다. - express-winston : Winston 로깅 라이브러리를 Express와 통합하여 요청 및 오류를 로깅하는 미들웨어 패키지로, 로그 관리를 효율적으로 수행할 수 있습니다. */
